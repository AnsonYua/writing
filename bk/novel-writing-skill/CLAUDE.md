# 网文长篇小说创作 Skill

专业的网文创作辅助工具，通过文档化管理解决长篇创作中的记忆衰减问题。提供从市场调研到章节创作的完整工作流程，确保50-60万字长篇的内容一致性和人物连贯性。

## 核心功能

- **7阶段创作流程**: 市场调研 → 风格确认 → 小说设定 → 故事大纲 → 角色开发 → 章节细纲（可选）→ 章节创作
- **防遗忘机制**: 分层记忆策略（短期/中期/长期）+ 强制检查清单
- **质量监控**: AI味检测、水文检测、内容一致性检查、人物/情节/氛围检测
- **交互式引导**: 使用 AskUserQuestion 实现点选式风格确认
- **文档模板**: 10个标准化模板，涵盖设定、角色、大纲、细纲、章节、追踪

## 重要提示

**每次章节创作前必须执行4步检查**：
1. 读取核心文档：novel-setting.md、story-outline.md、character-profiles.md、character-relationships.md
2. 读取进度追踪：progress-tracking.md（最近2-3章摘要）
3. 读取最近章节：chapter_XXX.md（最近2-3章完整内容）
4. 检查待回收伏笔：从progress-tracking.md中查看急需回收的伏笔

**创作后必须执行**：
1. 立即更新 progress-tracking.md（章节摘要、伏笔、角色变化）
2. 如有设定变化，同步更新对应文档（novel-setting.md、character-profiles.md等）
3. 定期（每5-10章）运行质量检测

## 关键创新

- **先大纲后角色**: 角色服务于情节，确保故事完整性
- **AskUserQuestion交互**: 风格确认使用点选式，避免繁琐问答
- **AI直接生成选项**: 每个询问都有"AI直接生成"和"跳过"选项
- **伏笔分级管理**: 高/中/长期伏笔，避免遗漏
- **人物关系图**: 使用Mermaid可视化追踪角色关系变化

## 适用场景

- 50-60万字的长篇网文创作
- 需要确保内容一致性的多章节连载
- 希望获得专业创作指导的作者
- 担心AI遗忘设定导致前后矛盾的情况

## 使用方式

在用户项目中创建 `novel-writing-skill/` 目录，将所有文件放入。用户说 "/开始创作" 开始流程，按照7个阶段逐步完成创作。每次章节创作时，AI必须先读取相关文档确保连贯性。
