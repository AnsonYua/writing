# 写作风格控制与AI味检测

## 第一部分:风格选择与确认

### 用户情况识别与处理

当用户输入 `/风格确认` 时,AI需要根据用户意图判断属于哪种情况:

#### 情况判断逻辑

**情况1:用户基于扫榜结果自主选择**
```
触发条件:
- 用户明确说出题材名称(如"我想写苟道流修仙")
- 用户引用扫榜报告中的具体作品或热点

处理流程:
1. 确认用户选择的题材
2. 立即询问写作风格偏好(6种风格选项)
3. 生成style-guide.md
4. 进入下一步:/小说设定
```

**情况2:用户有明确想法,不受扫榜影响**
```
触发条件:
- 用户直接说"我想写XXX"(明确题材)
- 用户表示"我已经想好了"

处理流程:
1. 确认用户选择的题材
2. 询问写作风格偏好(6种风格选项)
3. 生成style-guide.md
4. 进入下一步:/小说设定
```

**情况3:用户需要AI引导选择** ⭐重点
```
触发条件:
- 用户说"不知道"、"不确定"、"你帮我选"
- 用户表示"看不懂"、"不太了解"
- 用户询问"哪个好写"、"哪个受欢迎"

处理流程:
进入【3轮询问引导流程】(详见下方)
```

### AskUserQuestion 调用规范

#### 重要限制

1. **选项数量**: 每个问题最多4个选项（AskUserQuestion工具限制）
2. **multiSelect**: 大部分情况为False（单选），只有特殊需求才用True
3. **header**: 简短的分类标题（不超过12字）
4. **label**: 选项名称（简短，1-5个词）
5. **description**: 选项描述（详细说明，帮助用户理解）

#### 返回值处理

```python
# AskUserQuestion 返回格式
{
    "您希望在哪个平台发布作品？": "番茄小说"
}

# AI 需要提取并保存用户选择
platform_choice = response["您希望在哪个平台发布作品？"]
```

#### 错误处理

如果AskUserQuestion调用失败或用户中断：
- 优雅降级到文本问答模式
- 提示用户："点选界面暂时不可用，请直接输入您的选择"

### 6种风格选项

#### A. 个人风格复刻
**定义**:分析用户已有作品,模仿其写作特点

**适用场景**:
- 用户有明确的个人风格
- 想保持创作的连贯性
- 有可供参考的作品样本

**操作流程**:
1. 引导用户上传1-3篇作品样本(每篇1000-3000字)
2. 分析样本的:
   - 句式特点(长句多还是短句多/复合句比例)
   - 用词习惯(文艺词汇/口语词汇/专业术语)
   - 修辞手法(比喻/排比/反问的使用频率)
   - 叙述节奏(快节奏/慢节奏/张弛有度)
   - 对话风格(简洁/啰嗦/有特色口头禅)
3. 生成style-guide.md,记录分析结果
4. 后续创作严格遵循style-guide.md

**注意事项**:
- 样本需要是用户原创,不是他人作品
- 样本质量会直接影响模仿效果
- 如果用户风格本身有问题(如过度AI味),需要提醒

#### B. 人性化语言
**定义**:更接近自然人类表达,减少公式化内容

**特点**:
- 避免过于规整的句式结构
- 增加口语化表达
- 适当的语序变化和跳跃
- 减少"首先、其次、最后"等公式词

**写作要点**:
```
避免:
"首先,主角来到了市场。其次,他遇到了一个老人。最后,他得到了宝物。"

改为:
"市场人声鼎沸。主角挤过人群,突然被一个老人拉住。就这样,他得到了那件改变命运的宝物。"
```

**适用题材**:所有题材

#### C. 幽默风格
**定义**:增加诙谐元素和生动表达

**特点**:
- 使用搞笑的比喻和夸张
- 主角或配角有幽默感
- 情节中穿插轻松时刻
- 对话中有机智的抖机灵

**写作要点**:
```
普通写法:
"他跑得很快。"

幽默写法:
"他跑得像被债主追的赌徒,又像偷了骨头的二哈。"
```

**注意事项**:
- 幽默要适度,不要变成搞笑文
- 严肃场景不能随便插科打诨
- 要符合人物性格(不是所有人都会搞笑)

**适用题材**:都市生活、轻松向作品、部分玄幻

#### D. 模仿特定作家
**定义**:模仿知名网文作家的写作风格

**常见选择**:
- **猫腻**:细腻的心理描写,哲理性的对话,慢节奏铺陈
- **唐家三少**:热血战斗,能力体系清晰,节奏明快
- **我吃西红柿**:升级流,设定完整,爽点密集
- **烽火戏诸侯**:江湖气息浓厚,人物立体,对话精彩
- **爱潜水的乌贼**:设定严谨,伏笔众多,反转精彩

**操作方式**:
1. 用户指定作家名字
2. 分析该作家的典型特征
3. 在创作中模仿其风格
4. 提醒:是借鉴不是抄袭

**注意事项**:
- 只能借鉴风格,不能抄袭情节
- 要结合用户自己的故事,不是生搬硬套

#### E. 市井烟火风
**定义**:接地气,有生活气息,口语化强

**特点**:
- 大量生活化细节
- 方言和俗语的适度使用
- 平民视角,不高高在上
- 人物对话生活化

**写作要点**:
```
文艺风:
"夕阳西下,晚霞如血,染红了天际。"

市井烟火风:
"天快黑了,菜市场的大妈们开始甩卖白菜,5毛一斤,不要钱似的。"
```

**适用题材**:都市生活、现实题材、部分穿越文

#### F. 文青清新风
**定义**:细腻感性,意境优美

**特点**:
- 文字优美,有诗意
- 情感细腻,心理描写深入
- 环境描写有意境
- 适度使用比喻和排比

**写作要点**:
```
直白风:
"她很伤心。"

文青清新风:
"她站在雨里,分不清脸上的是雨水还是泪水。那一刻,整个世界都安静了,只剩下雨声,和心碎的声音。"
```

**注意事项**:
- 不要过度抒情,影响节奏
- 网文读者更喜欢情节,不要本末倒置
- 适度使用,点到为止

**适用题材**:言情、青春、部分古言

## 第二部分:AI味检测

### AI味的5大特征

#### 1. 过于规整的段落结构
**表现**:
- 每段长度几乎一致
- 段落开头的句式相似
- 缺乏长短变化

**示例**:
```
❌ AI味重:
"主角来到了市场。市场里人很多。他看到了一个老人。老人给了他一件宝物。"
(每句长度相似,结构单一)

✅ 改进后:
"市场人声鼎沸。主角好不容易挤过人群,突然被一只干瘦的手拉住。是个老人,浑浊的眼睛盯着他,手里攥着一个锦盒。"
(句子长短不一,结构多变)
```

#### 2. 重复性表达和固定句式
**表现**:
- 过多"首先、其次、最后"
- 过多"不仅...而且..."
- 过多"然而"、"因此"等连接词
- 相同的开头方式

**示例**:
```
❌ AI味重:
"首先,主角提升了实力。其次,他获得了宝物。最后,他打败了敌人。"

✅ 改进后:
"三天闭关,主角的实力突飞猛进。正要出关时,一道金光从天而降——那是他梦寐以求的神器。有了这个,那个嚣张的敌人,该颤抖了。"
```

#### 3. 过度完美的逻辑性
**表现**:
- 每个段落都有主题句
- 过渡太过流畅自然
- 缺乏跳跃感和意外

**示例**:
```
❌ AI味重:
"由于天气很好,所以主角决定出门。因为要出门,所以他换了衣服。换好衣服后,他出发了。"
(逻辑链太完整,像作文)

✅ 改进后:
"难得的好天气。主角看了眼镜子,换上那件洗了三次还没干透的T恤,出门。"
(省略了一些逻辑过渡,更接近人的思维)
```

#### 4. 缺乏个人情感色彩
**表现**:
- 客观陈述多,主观评价少
- 缺乏态度和立场
- 像在陈述事实,不像在讲故事

**示例**:
```
❌ AI味重:
"主角进入了房间。房间里有一张床,一张桌子,一把椅子。"
(纯客观陈述,没感情)

✅ 改进后:
"主角推开门,破旧的房间扑面而来一股霉味。床上的被子皱巴巴的,桌子上积了一层灰。见鬼,这就是他花了三个月工资租的'精装修'?"
(有情感,有态度,有讽刺)
```

#### 5. 词汇多样性不足
**表现**:
- 频繁使用相同词汇
- "非常"、"很"、"特别"等程度副词过多
- 缺乏同义词替换

**示例**:
```
❌ AI味重:
"他很生气。她很伤心。这件事很重要。"
(很很很...)

✅ 改进后:
"他气得砸了杯子。她哭到眼睛红肿。这件事,关乎所有人的生死。"
(用动作和结果代替程度副词)
```

## 第三部分:风格保持策略

### 创作过程中的风格控制

**每章创作前**:
- 重温style-guide.md的核心要点
- 回顾最近1-2章的语言风格
- 确保新章节风格连贯

**创作过程中**:
- 定期对照style-guide.md自检
- 避免跑偏回到AI化表达
- 保持风格的持续性

**创作完成后**:
- 通读全章,检查风格一致性
- 用/检查AI味功能进行检测
- 根据反馈调整优化

### 风格与情节的平衡

**原则**:
- 风格服务于内容,不能本末倒置
- 关键情节处可以风格化更强
- 紧张情节处语言要简洁有力
- 抒情段落可以风格化表达

**示例**:
```
紧张场景(语言简洁):
"剑光闪过。血飞溅。一秒,生死已分。"

抒情场景(可以文艺):
"夕阳西沉,如同烧尽的薪火。她站在山巅,任风吹乱长发,眼中是化不开的悲伤。"
```
