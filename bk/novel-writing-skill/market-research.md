# 网文市场调研与扫榜方法

## 扫榜方式选择

用户在输入 `/扫榜引导` 后,需要选择扫榜方式:

### 方式1: 用户自己扫榜
**触发时机**:用户希望亲自了解市场,或对AI分析存疑时

**操作流程**:
1. AI提供扫榜平台链接和操作指南
2. 用户自行访问榜单并记录数据
3. 用户将扫榜结果反馈给AI
4. AI基于用户提供的数据给出分析建议

**优势**:
- 更深入地了解市场
- 建立自己的市场直觉
- 可以根据兴趣深入研究特定作品

### 方式2: AI协助联网扫榜 ⭐推荐
**触发时机**:用户希望快速获得最新市场数据和专业分析时

**操作流程**:
1. AI通过联网功能访问各大平台榜单
2. AI实时抓取最新榜单数据(起点/番茄/晋江等)
3. AI进行多维度分析(书名/简介/热点/趋势)
4. AI生成详细的扫榜报告
5. **自动保存**:AI将扫榜结果保存为 `market-research-YYYY-MM-DD.md` 文件

**优势**:
- 数据最新、实时抓取
- 节省用户时间
- AI提供专业的趋势分析
- 扫榜结果永久保存,方便后续查阅

**输出文档**:`market-research-YYYY-MM-DD.md`(与小说项目文件一起保存)

## 扫榜的核心目的

### 为什么要扫榜
- 了解当前热门小说类型和读者喜好
- 跟随市场趋势提高作品竞争力
- 学习成功作品的写作技巧和营销策略
- 避免闭门造车,创作出市场不接受的作品

### 扫榜的正确心态
- 不是照搬,而是学习规律
- 不是模仿,而是找到共性
- 不是放弃创意,而是在市场规律内创新
- 不是一次性工作,而是持续的市场监测

## 扫榜重点关注要素

### 1. 书名特点分析
**流行的命名方式**:
- 问句开头引发好奇:"侯府忘恩义?摄政王撑腰,不原谅"
- 热点+反转组合:"黄毛霸凌我?家叔祁同伟呀!"
- 身份+冲突模式:"退婚后,我成了全球首富"
- 系统+能力模式:"开局签到亿万资产"

**书名的核心作用**:
- 3秒内抓住眼球
- 暗示核心爽点
- 制造认知反差
- 触发读者好奇

**书名设计原则**:
- 简洁直观:10-20字最佳
- 包含热点元素:退婚、霸总、系统、重生等
- 制造矛盾或反差:弱者vs强者、过去vs现在
- 避免生僻字和复杂概念

### 2. 简介结构分析
**高效简介的标准结构**:
```
第一部分:卖点前置
[类型标签+核心吸引点]
例:"退婚流+系统金手指"

第二部分:矛盾核心
[主角处境+面临的困境+对手设定]
例:"被未婚妻退婚,家族要逐出门,京城豪门看不起"

第三部分:金手指展示
[主角获得的能力/优势/转机]
例:"觉醒全球首富系统,日进斗金,财富碾压一切"

第四部分:悬念钩子
[暗示主角将如何反击/展望]
例:"那些看不起我的人,终将跪在我面前!"
```

**简介的常见问题**:
- ❌ 过于复杂,看不懂核心卖点
- ❌ 过于文艺,不接地气
- ❌ 没有矛盾感,平淡无奇
- ❌ 金手指不够明确,读者不知道主角凭什么翻身

**优秀简介的特征**:
- ✅ 卖点明确,一眼看懂
- ✅ 矛盾尖锐,引发共鸣
- ✅ 金手指诱人,让人期待
- ✅ 语言接地气,不装腔作势

### 3. 开篇技巧分析
**前800字的黄金法则**:
- 第1-30字:必须抓住眼球(冲突/反差/悬念)
- 第31-300字:展示核心矛盾和主角处境
- 第301-800字:暗示转机或展示初始爽点

**成功开篇的常见模式**:
1. **冲突开局**:主角正在被欺负/羞辱/威胁
2. **反差开局**:看似弱小的主角展示意外能力
3. **悬念开局**:不同寻常的场景或事件
4. **爽点开局**:主角直接展示强大或特殊

**开篇的禁忌**:
- ❌ 长篇环境描写
- ❌ 主角自述身世背景
- ❌ 上帝视角的世界观科普
- ❌ 大段内心独白

### 4. 评分与评论分析
**评分的参考价值**:
- 8分以上:优质作品,值得深入学习
- 7-8分:及格线,可学习其优点
- 7分以下:可能存在明显问题

**评论的关注点**:
- 读者最喜欢什么(爽点、角色、情节)
- 读者最不满什么(注水、拖沓、人设崩)
- 读者追更的动力是什么
- 读者弃书的原因是什么

## 番茄小说扫榜操作步骤

### 电脑端操作
1. 访问番茄榜单:https://fanqienovel.com/library/all/page_1?sort=hottes
2. 榜单筛选:
   - 优先查看"热度榜"(代表当前流行趋势)
   - 筛选30万字以下作品(新书风向标)
   - 注意更新频率和追读数据

3. 信息记录:
   - 前10-20名作品的书名
   - 题材类型和核心卖点
   - 书名和简介的共同特点
   - 更新节奏和章节数

### 手机APP操作
1. 下载番茄小说APP
2. 查看完整简介:
   - 电脑端简介可能被截断
   - 手机端能看到完整版本
   - 记录简介的结构和亮点

3. 查看评分和评论:
   - 读者最喜欢的点
   - 读者吐槽的问题
   - 高赞评论的内容
   - 追更读者的反馈

4. 试读开篇:
   - 阅读前三章
   - 分析开篇的吸引力
   - 总结爽点分布节奏
   - 学习对话和描写风格

## 扫榜数据整理方法

### 建立分析表格
推荐使用Excel或Google Sheets,建立如下表格:

| 排名 | 书名 | 题材 | 核心卖点 | 金手指 | 书名特点 | 简介结构 | 开篇方式 | 评分 | 亮点 | 缺点 |
|------|------|------|----------|--------|----------|----------|----------|------|------|------|
| 1    | ... | ... | ...      | ...    | ...      | ...      | ...      | 8.5  | ... | ...  |
| 2    | ... | ... | ...      | ...    | ...      | ...      | ...      | 8.3  | ... | ...  |
| ...  | ... | ... | ...      | ...    | ...      | ...      | ...      | ...  | ... | ...  |

### 数据分析维度
**横向对比(同类型作品)**:
- 相同题材的前10名作品有什么共性?
- 书名模式有什么相似之处?
- 金手指设定有什么套路?
- 开篇的吸引方式是否类似?

**纵向对比(不同类型作品)**:
- 不同题材的爆点设计有什么不同?
- 哪种题材更受欢迎?
- 市场热度的变化趋势?

**时间对比(定期扫榜)**:
- 本月和上月的榜单变化
- 新出现的热点元素
- 衰退的套路和模式

## 扫榜后的行动建议

### 1. 确定创作方向
基于扫榜结果,回答以下问题:
- 我的作品属于哪个题材?
- 这个题材当前的市场热度如何?
- 我的核心卖点是什么?
- 我的金手指设定有吸引力吗?
- 我的开篇能在前800字抓住读者吗?

### 2. 设计书名和简介
**书名设计**:
- 参考同类型成功作品的书名模式
- 确保包含核心卖点元素
- 制造矛盾或反差感
- 测试:让朋友看书名,3秒内能否理解卖点?

**简介设计**:
- 采用"卖点前置+矛盾核心+金手指展示+悬念钩子"结构
- 语言接地气,不文艺腔
- 每句话都有信息量
- 测试:让朋友看简介,是否产生阅读兴趣?

### 3. 规划开篇结构
**前三章的重要性**:
- 第1章前800字:决定读者是否继续看
- 第1-3章:决定读者是否追更
- 番茄等平台的推荐机制高度依赖前期留存率

**开篇规划要点**:
- 第1章:强烈冲突+初始爽点
- 第2章:展示金手指+主角特质
- 第3章:初步反击+留下悬念

### 4. 学习成功作品的节奏
**追读几部同类型高分作品**:
- 爽点分布的频率(多久一个爽点)
- 矛盾升级的节奏(如何逐步加压)
- 伏笔埋设的技巧(什么时候埋,什么时候收)
- 情感渲染的火候(什么时候煽情,什么时候克制)

## 常见扫榜误区

### 误区1:照搬成功作品
- ❌ 错误:直接抄袭情节和人设
- ✅ 正确:学习其中的规律和套路,用自己的方式表达

### 误区2:追逐所有热点
- ❌ 错误:今天写退婚,明天写霸总,后天写系统
- ✅ 正确:选择1-2个自己擅长且有市场的方向深耕

### 误区3:完全放弃创意
- ❌ 错误:认为市场化=套路化=没创意
- ✅ 正确:在市场规律内寻找创新空间,做"熟悉的陌生感"

### 误区4:只看榜单不看评论
- ❌ 错误:只记录书名和简介,不深入了解读者反馈
- ✅ 正确:评论是最直接的市场反馈,比数据更有价值

### 误区5:一次扫榜终身受用
- ❌ 错误:扫一次榜就不再关注市场
- ✅ 正确:市场在变化,热点在更新,需要定期扫榜(建议每月1次)

## 不同平台的扫榜差异

### 番茄小说(字节跳动)
**特点**:
- 男频为主,重爽点密度
- 免费阅读,靠广告分成
- 推荐算法激进,新书有流量机会
- 读者画像:下沉市场,喜欢直接爽快的内容

**扫榜侧重**:
- 开篇前3章的吸引力
- 爽点分布频率(建议每章2-3个小爽点)
- 金手指的直接性(不要太隐晦)

### 起点中文网(阅文集团)
**特点**:
- 付费阅读,重长期粘性
- 读者付费意愿强,审美更成熟
- 经典题材更受欢迎
- 读者画像:老书虫,能接受复杂设定

**扫榜侧重**:
- 世界观的完整性
- 人物塑造的深度
- 情节的逻辑性
- 文笔和叙事能力

### 晋江文学城
**特点**:
- 女频为主,重情感线
- 付费阅读,读者忠诚度高
- 注重CP感和情感渲染
- 读者画像:女性读者,喜欢甜宠、虐恋、宫斗

**扫榜侧重**:
- 男女主的人设吸引力
- 情感线的铺陈和爆发
- 虐点和甜点的节奏控制
- 对话的张力和暧昧感

## AI协助扫榜执行流程

### 第一步:确认用户需求
当用户输入 `/扫榜引导` 时,AI需要询问:

```
您希望采用哪种扫榜方式?

1. **用户自己扫榜** - 我提供平台链接和操作指南,您亲自扫榜
2. **AI协助扫榜** ⭐推荐 - 我联网获取最新榜单并生成详细分析报告

请选择 1 或 2:
```

### 第二步:执行AI扫榜(如果选择方式2)

**2.1 联网获取榜单数据**
AI通过WebSearch/WebFetch工具访问:
- 起点中文网: https://www.qidian.com/rank/
- 番茄小说: https://fanqienovel.com/library/all/page_1?sort=hottest
- 晋江文学城: https://www.jjwxc.net/topten.php

**2.2 抓取关键信息**
对于每个平台,记录:
- TOP10-20作品的书名
- 作者名称
- 题材类型
- 月票/评分/追读数据
- 核心卖点(从简介提取)
- 金手指设定(如有)

**2.3 多维度分析**
对抓取的数据进行分析:
- 书名命名规律总结
- 热门题材排行
- 新兴热点识别
- 衰退趋势标注
- 平台差异对比

**2.4 生成扫榜报告**
报告包含以下部分:
```markdown
# 网文市场扫榜报告
**日期**: YYYY-MM-DD
**平台**: 起点/番茄/晋江

## 一、各平台TOP10榜单
[具体榜单数据]

## 二、2025年热点趋势分析
### 正在上升的热点
### 衰退的热点
### 平台特色热点

## 三、书名命名规律
[总结最新的书名公式]

## 四、创作建议
### 男频创作建议
### 女频创作建议

## 五、标杆作品推荐
[列出值得深入学习的作品]
```

### 第三步:保存扫榜结果

**3.1 生成文件**
文件命名:`market-research-YYYY-MM-DD.md`
文件位置:用户指定的项目目录(默认为小说项目根目录)

**3.2 文件内容**
完整的扫榜报告,包括:
- 所有抓取的榜单数据
- 热点分析结果
- 创作建议
- 数据来源链接

**3.3 询问用户保存位置**
```
扫榜报告已生成!

请告诉我您希望将报告保存到哪个目录?
(例如:/Users/xxx/Documents/我的小说/)

如果不指定,将保存到当前工作目录。
```

**3.4 创建文件**
使用Write工具创建文件,并告知用户保存成功:
```
✅ 扫榜报告已保存到: [文件路径]

您可以随时查看这份报告,它将与您的小说项目文件一起保存。
建议每月重新扫榜一次,以跟踪市场变化。
```

### 第四步:基于扫榜结果引导用户

扫榜报告保存后,询问用户下一步:

```
基于扫榜结果,现在请告诉我:

1. 您想写**男频**还是**女频**?
2. 您偏好哪个**题材**?(根据扫榜报告的热点选择)
3. 或者您有**自己的想法**,不受热点限制?

确定后,我们将进入: `/风格确认`
```
