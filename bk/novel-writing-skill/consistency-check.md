# 内容一致性检查清单

## 使用说明

本文档是长篇小说创作中最重要的质量保障工具。由于长篇创作可能持续数月甚至数年,很容易出现前后矛盾的情况。通过定期使用本清单检查,可以避免严重的一致性问题。

**建议检查频率**:
- 每5-10章进行一次全面检查
- 重要情节转折前必须检查
- 读者指出问题时立即检查

---

## 第一类:世界观设定一致性

### 1. 基础世界观
- [ ] 世界的基本规则(物理定律/魔法规则/修炼体系等)前后一致
- [ ] 时代背景(古代/现代/未来)设定一致
- [ ] 地理设置(国家/城市/地名)前后一致
- [ ] 社会结构(阶级/职业/组织)没有矛盾

**检查方法**:
对比novel-setting.md中的设定与已写章节,确认无冲突

**常见问题**:
```
第5章:这个世界没有魔法
第30章:主角学会了一个魔法
→ 严重矛盾,需立即修正
```

### 2. 力量体系
- [ ] 等级划分清晰且一致(如修真等级/职业等级)
- [ ] 力量表现前后一致(不能今天秒天秒地,明天被小兵打败)
- [ ] 升级速度合理(不能突然从弱鸡变大佬)
- [ ] 能力限制保持一致(不能今天能飞,明天忘了能飞)

**检查方法**:
列出主角及主要角色的实力变化轨迹,确认合理性

**常见问题**:
```
第10章:主角筑基期,打不过金丹期
第11章:主角突破金丹,轻松秒杀元婴期
→ 力量体系崩坏,破坏读者认知
```

### 3. 重要设定
- [ ] 金手指的规则保持一致(系统规则/空间能力/特殊血脉等)
- [ ] 重要物品的属性一致(神器能力/丹药效果等)
- [ ] 组织势力的设定一致(门派规矩/公司制度等)
- [ ] 特殊能力的限制一致(使用代价/冷却时间等)

---

## 第二类:人物设定一致性

### 1. 基础信息
- [ ] 姓名拼写一致(不能一会儿李明,一会儿李名)
- [ ] 年龄前后合理(随时间推移合理增长)
- [ ] 外貌特征一致(眼睛颜色/身高/特殊标记等)
- [ ] 身份职业一致(不能无故改变)

**检查方法**:
对比character-profiles.md与已写章节,逐个角色检查

**常见问题**:
```
第3章:王强是个胖子,体重200斤
第20章:王强身材精瘦,动作灵活
→ 除非特别说明减肥,否则矛盾
```

### 2. 性格特征
- [ ] 核心性格保持稳定(不能前面谨慎,后面莽撞)
- [ ] 性格发展有合理轨迹(成长/黑化要有过程)
- [ ] 说话方式风格一致(口头禅/用词习惯等)
- [ ] 行为模式符合性格(决策/反应方式等)

**检查方法**:
回顾角色的关键行为,确认符合性格设定

**常见问题**:
```
设定:主角性格冷静理智,从不冲动
第50章:主角被激怒,不顾一切冲上去
→ 需要给出合理理由(如涉及底线/至亲受辱等)
```

### 3. 人物关系
- [ ] 人物关系状态与剧情发展一致
- [ ] 关系变化有明确原因和过程
- [ ] 称呼前后一致(不能一会儿"你",一会儿"您")
- [ ] 亲疏远近符合剧情发展

**检查方法**:
画出人物关系图,标注重要变化节点

**常见问题**:
```
第10章:主角和A已经决裂,老死不相往来
第15章:主角和A亲密无间,像好兄弟
→ 缺少和解过程,读者困惑
```

### 4. 人物能力
- [ ] 能力设定前后一致
- [ ] 能力成长有合理过程
- [ ] 不能突然遗忘已有能力
- [ ] 不能突然获得不合理的能力

**检查方法**:
为主要角色建立能力清单,逐章更新

**常见问题**:
```
第20章:主角学会了空间瞬移
第30章:被敌人包围,主角完全忘了瞬移能力
→ 为了情节需要强行降智,读者不满
```

## 第三类:情节逻辑一致性

### 1. 时间线
- [ ] 事件发生的时间顺序清晰
- [ ] 时间推进合理(不能今天还是夏天,明天就冬天了,除非特殊原因)
- [ ] 人物年龄增长符合时间线
- [ ] 重要时间节点(节日/约定时间等)前后一致

**检查方法**:
制作时间线表格,标注关键事件

**常见问题**:
```
第5章:现在是5月初,距离大比还有三个月
第8章:现在是6月,大比已经结束了
→ 时间推进混乱
```

### 2. 因果关系
- [ ] 事件之间有合理的因果链条
- [ ] 人物行动有明确动机
- [ ] 结果符合前面的铺垫
- [ ] 没有突兀的情节转折

**检查方法**:
列出主要情节点,检查因果逻辑

**常见问题**:
```
前文:主角千方百计躲避敌人
后文:主角突然主动送上门
→ 缺少合理的动机转变过程
```

### 3. 信息一致性
- [ ] 已透露的信息保持一致
- [ ] 不能矛盾地揭示真相
- [ ] 人物的已知信息和行动匹配
- [ ] 伏笔回收与前文铺垫一致

**检查方法**:
记录所有重要信息透露点,交叉检查

**常见问题**:
```
第15章:揭示凶手是A
第40章:突然说其实凶手是B,A是替罪羊
但前文没有任何A被冤枉的线索
→ 强行反转,缺乏铺垫
```

## 第四类:场景细节一致性

### 1. 地点描述
- [ ] 重要地点的描述前后一致
- [ ] 地点间的距离/方位关系合理
- [ ] 建筑/房间布局前后一致
- [ ] 地点特征描写一致

**检查方法**:
为重要场所绘制简易地图

**常见问题**:
```
第10章:主角的房间在三楼
第25章:主角从房间窗户跳到一楼院子
→ 三楼到一楼的距离矛盾(前文未说明楼层很矮)
```

### 2. 物品细节
- [ ] 重要物品的外观描述一致
- [ ] 物品的功能/属性一致
- [ ] 物品的归属保持追踪
- [ ] 物品不能无故消失或出现

**检查方法**:
建立重要物品清单,追踪其流向

**常见问题**:
```
第5章:主角获得了一把神剑
第50章:神剑从未再提起,不知所踪
→ 重要物品不能忘记
```

---

## 第五类:伏笔与回收一致性

### 1. 伏笔记录
- [ ] 所有埋下的伏笔都有记录
- [ ] 记录包含:章节位置/内容/计划回收时机
- [ ] 定期检查待回收伏笔
- [ ] 不遗漏任何未回收伏笔

**检查方法**:
在progress-tracking.md中维护伏笔清单

**示例记录**:
```markdown
## 待回收伏笔
1. [第12章] 神秘老人说的预言 → 计划在第50章左右揭晓
2. [第20章] 主角父亲的遗物盒子 → 计划在第60章打开
3. [第35章] 配角A的可疑行为 → 计划在第45章揭示身份
```

### 2. 回收质量
- [ ] 回收时与伏笔内容一致
- [ ] 回收时机合理,不过早不过晚
- [ ] 回收方式令人满意(不敷衍)
- [ ] 回收后删除伏笔记录中的该项

## 快速检查流程

### 每章完成后的快速检查(5分钟)
1. 对照character-profiles.md,确认出场角色的基本信息无误
2. 检查本章人物行为是否符合性格设定
3. 检查本章时间线是否合理
4. 检查是否使用了与设定矛盾的能力或规则
5. 在progress-tracking.md中记录本章要点和新埋伏笔

### 每5-10章的深度检查(30分钟)
1. 对比novel-setting.md,确认世界观无矛盾
2. 对比character-profiles.md,确认所有角色的信息一致
3. 检查时间线表,确认事件顺序合理
4. 检查伏笔清单,规划回收时机
5. 检查力量体系,确认升级合理
6. 重读最近章节,检查是否有前后矛盾

### 重大情节前的全面检查(1-2小时)
1. 完整重读所有基础文档(setting/characters/outline)
2. 检查所有一致性清单项目
3. 确认即将发生的情节与前文无矛盾
4. 检查相关角色的状态和能力
5. 确认所有涉及的伏笔和铺垫
6. 如需要,更新相关文档

---

## 常见一致性错误TOP10

1. **角色外貌前后不一**:眼睛颜色/身高/特征变化
2. **力量体系崩坏**:战力忽高忽低,无法自洽
3. **性格突变**:前后性格判若两人,无合理过程
4. **时间线混乱**:事件顺序矛盾,时间推进不合理
5. **能力遗忘**:主角忘记已经学会的能力
6. **信息矛盾**:前后透露的信息相互冲突
7. **地理矛盾**:距离/方位/场景描述不一致
8. **伏笔遗漏**:埋下的伏笔从未回收
9. **人物关系混乱**:亲疏远近突然改变
10. **物品消失**:重要物品无故不见

---

## 修正指南

### 发现矛盾后的处理流程

**步骤1:评估严重程度**
- 严重矛盾(影响主线情节):必须立即修正
- 中等矛盾(影响支线或角色):尽快修正
- 轻微矛盾(细节问题):可以在修订时处理

**步骤2:确定修正方案**
- 方案A:修改新章节,使其符合旧设定
- 方案B:修改旧章节,使其符合新发展
- 方案C:在中间章节加入解释或过渡

**步骤3:执行修正**
- 修改相关章节
- 更新相关文档(setting/characters等)
- 在progress-tracking.md中记录修正内容

**步骤4:通知读者(如果已发布)**
- 严重矛盾:发布修正说明,重新上传章节
- 中等矛盾:作家的话中说明
- 轻微矛盾:下次修订时处理
