# 质量控制与水文检测

## 第一部分:水文控制

### 什么是水文

**定义**:
水文是指对推动剧情没有实质作用的冗余内容,包括:
- 过度的环境描写
- 无意义的对话
- 重复的内心独白
- 无关的回忆闪回
- 拖沓的场景铺陈

**危害**:
- 拖慢节奏,读者流失
- 稀释爽点密度
- 影响阅读体验
- 降低订阅意愿(付费平台尤其致命)

---

### 水文控制5大原则

#### 原则1:主线剧情占比原则
**标准**:每章至少70%的内容直接推动主线剧情发展

**判断方法**:
- 这段内容删除后,剧情是否受影响?
- 这段内容对主角的目标推进有帮助吗?
- 这段内容展现了人物成长或关系变化吗?

**示例对比**:
```
❌ 水文示例:
"主角走在大街上。街道两旁种满了梧桐树,树叶在微风中沙沙作响。
阳光透过树叶的缝隙洒下斑驳的光影。远处传来小贩的叫卖声。
路边的商店橱窗里摆满了各种商品。主角看了看,继续往前走。"
→ 问题:500字环境描写,对剧情毫无推动

✅ 改进后:
"主角快步走过梧桐树荫,一边走一边翻看手机里的那条短信——
'今晚8点,老地方,我有重要的事告诉你。'
是她。三年没联系,突然约见面,什么事这么急?"
→ 优势:环境一笔带过,重点是推动剧情的短信
```

#### 原则2:环境描写控制原则
**标准**:环境描写不超过每章总内容的15%

**使用时机**:
- 新场景首次出现(必要交代)
- 环境对情节有直接影响(如:密室/战场)
- 环境烘托气氛(如:悬疑/恐怖场景)
- 环境暗示信息(如:破败的家暗示贫穷)

**禁止情况**:
- ❌ 为了凑字数的大段环境描写
- ❌ 与剧情无关的风景描写
- ❌ 已经描写过的场景反复描写
- ❌ 过于细致的无关细节

**高效环境描写**:
```
低效(200字):
"房间很大,大约有50平米。墙壁是白色的,地板是棕色的木地板。
房间里有一张双人床,床单是蓝色的,很干净。床头柜上放着一盏台灯...
[继续描述家具摆设200字]"

高效(30字):
"房间出奇的整洁,不像是有人常住的样子。床单连褶皱都没有,透着一股冷清。"
→ 30字既交代了环境,又暗示了人物状态
```

#### 原则3:对话效率原则
**标准**:对话必须高效推动情节或揭示人物性格

**每句对话必须满足以下至少一项**:
1. 推动剧情(透露信息/制造冲突/引发行动)
2. 展现性格(通过说话方式体现人物特征)
3. 增进/破坏关系(改变人物关系状态)
4. 制造悬念(引发读者好奇)

**禁止的无效对话**:
- ❌ 客套寒暄:"你好!""你好!""最近怎么样?""还行。"
- ❌ 无意义闲聊:"今天天气不错。""是啊。"
- ❌ 重复已知信息:"我们要去xxx。""是的,我们要去xxx。"

**示例对比**:
```
❌ 无效对话:
"早上好,李明。"
"早上好,王强。"
"今天天气不错啊。"
"是啊,挺好的。"
"你吃早饭了吗?"
"吃了,你呢?"
"我也吃了。"
→ 问题:7行对话,0信息量

✅ 高效对话:
"李明,昨晚的事处理完了?"
王强压低声音:"处理了,但有个麻烦——有人看见了。"
李明脸色一变:"谁?"
"你最好希望不是她。"
→ 优势:4行对话,推动剧情+制造悬念+展现紧张气氛
```

#### 原则4:内心独白控制原则
**标准**:内心独白不超过每章总内容的10%

**使用时机**:
- 重大决策前的心理斗争
- 情感爆发的关键时刻
- 揭示隐藏动机或秘密
- 表现人物成长或顿悟

**禁止情况**:
- ❌ 长篇大论的自我分析
- ❌ 重复已经表现过的想法
- ❌ 可以用动作/对话表现的内容
- ❌ 流水账式的思考过程

**示例对比**:
```
❌ 冗长独白:
"我该怎么办呢?如果我选择A,会怎样呢?会有好处,但也有坏处。
如果我选择B呢?B的好处是什么?坏处又是什么?我要仔细分析一下。
A的好处有三点:第一...第二...第三...
A的坏处也有三点:第一...第二...第三...
B的好处和坏处...[继续分析500字]"
→ 问题:读者不想看你的思考过程,只想看结果

✅ 简洁独白:
"A还是B?他攥紧拳头。该死,两条路都是死路。
但至少...A能保住她。"
→ 优势:寥寥数语,既展现纠结,又暗示决定
```

#### 原则5:回忆内容限制
**标准**:每章回忆内容不超过5%

**使用时机**:
- 解释当前行为的动机(必要的背景)
- 揭示关键伏笔(之前埋下的信息)
- 人物创伤或执念的展现(推动性格塑造)
- 与当前情节直接相关的过往

**禁止情况**:
- ❌ 无关紧要的童年回忆
- ❌ 冗长的人生经历回顾
- ❌ 与当前情节无关的闪回
- ❌ 重复的回忆内容

**示例对比**:
```
❌ 冗余回忆:
"看着眼前的这一幕,主角想起了小时候。那时候他还是个孩子。
记得有一年夏天,他和小伙伴们去河边玩耍...[回忆整个童年300字]
这些往事,让他感慨万千。"
→ 问题:与当前情节无关,纯凑字数

✅ 简洁回忆:
"这熟悉的场景让他想起了什么——三年前,就是这里,师父临终前说的那句话。
原来是这个意思!"
→ 优势:回忆直接服务于当前情节,并推动理解
```

### 水文自检流程

**触发指令**:/水文自检

**检测步骤**:

1. 统计主线剧情占比
2. 检查环境描写比例
3. 评估对话有效性
4. 统计内心独白比例
5. 检查回忆内容比例
6. 生成检测报告

**检测报告格式**:
```markdown
# 水文检测报告

## 章节统计
- 总字数:XXXX字
- 主线剧情:XXXX字(XX%)
- 环境描写:XXXX字(XX%)
- 对话部分:XXXX字(XX%)
- 内心独白:XXXX字(XX%)
- 回忆闪回:XXXX字(XX%)

## 问题识别

### 主线占比不达标
当前:XX% (标准:≥70%)
**问题段落**:
"[引用水文段落]"

**建议**:
删减或重写以上段落,将重点放在推动剧情上。

### 环境描写超标
当前:XX% (标准:≤15%)
**过度描写位置**:
- 第X段:[简述]
- 第X段:[简述]

**建议**:
保留关键环境信息,删除与剧情无关的细节。

### 无效对话过多
识别到X处无效对话:
1. "你好""你好"[第X行]
2. [其他无效对话]

**建议**:
每句对话必须推动剧情或展现性格,删除客套寒暄。

### 其他问题
[列出其他发现的问题]

## 整体评分:X/10
(10分=无水文,1分=严重水文)

## 改进优先级
1. [最严重的问题]
2. [次要问题]
3. [其他问题]
```

## 第二部分:多维度质量检测

### 内容一致性检测

**触发指令**:/内容一致性

**检测维度**:

#### 1. 逻辑一致性
- 情节发展的因果关系是否合理
- 时间线是否存在矛盾
- 人物行为动机是否前后一致
- 世界设定规则是否自洽

**示例**:
```
矛盾示例:
第3章:"主角不会武功"
第5章:"主角使出家传剑法"
→ 问题:设定前后矛盾

修正方法:
- 方法1:第3章改为"主角武功很弱"
- 方法2:第4-5章之间加入"主角回忆起家传剑法"的情节
```

#### 2. 细节一致性
- 人物外貌特征描写是否一致
- 人物说话习惯是否一致
- 场景描写是否前后呼应
- 重要道具的描述是否一致

**示例**:
```
矛盾示例:
第2章:"她有一双蓝色的眼睛"
第10章:"她棕色的眼睛闪着光芒"
→ 问题:外貌描写矛盾

修正方法:
统一为一种颜色,或者解释原因(如戴了美瞳)
```

---

### 人物塑造检测

**触发指令**:/人物塑造

**检测维度**:

#### 1. 人物行为合理性
- 行为是否符合人设
- 决策是否符合性格
- 情感反应是否真实
- 成长轨迹是否自然

**示例**:
```
不合理示例:
设定:主角性格谨慎小心
行为:主角不做任何调查就冲进敌人老巢
→ 问题:行为与性格矛盾

修正方法:
- 给出合理理由(如:有人被抓,来不及调查)
- 或修改行为(加入调查和准备过程)
```

#### 2. 对话真实性
- 是否符合人物身份和文化程度
- 是否符合当前场景和情绪
- 是否体现人物特色(口头禅/说话习惯)
- 是否推动情节或展现性格

---

### 情节控制检测

**触发指令**:/情节控制

**检测维度**:

#### 1. 节奏把控
- 重要情节是否铺垫充分
- 高潮迭起是否合理
- 是否存在节奏断档(过快/过慢)
- 悬念设置是否适当

#### 2. 剧情完整性
- 主要情节线是否完整
- 次要情节线是否收束
- 伏笔是否合理回收
- 结局是否合理交代

**示例**:
```
伏笔遗漏示例:
第5章:埋下伏笔"那个神秘人物是谁?"
第50章:故事结束,从未解释
→ 问题:伏笔未回收,读者不满

修正方法:
- 在适当章节揭晓身份
- 或在进度追踪中标注待回收伏笔,提醒自己
```

---

### 氛围营造检测

**触发指令**:/氛围营造

**检测维度**:

#### 1. 场景描写
- 环境描写是否到位且不过度
- 氛围渲染是否恰当
- 细节描写是否生动
- 意境营造是否成功

#### 2. 情感渲染
- 情感递进是否自然
- 感情表达是否真实
- 情绪调动是否恰当
- 共情效果是否达到

## 质量控制检查清单

### 创作前检查
- [ ] 已明确本章推动的主线剧情
- [ ] 已规划本章的关键情节点
- [ ] 已确定需要出场的角色
- [ ] 已确认与前文的衔接点

### 创作过程中检查
- [ ] 每段内容都对剧情有推动作用
- [ ] 环境描写控制在合理范围
- [ ] 对话高效且有信息量
- [ ] 内心独白简洁必要
- [ ] 回忆内容与当前情节相关

### 创作后检查
- [ ] 主线剧情占比≥70%
- [ ] 环境描写≤15%
- [ ] 内心独白≤10%
- [ ] 回忆内容≤5%
- [ ] 无明显的逻辑矛盾
- [ ] 人物行为符合设定
- [ ] 已为下章留下钩子

### 定期检查(每5-10章)
- [ ] 主要情节线推进正常
- [ ] 次要情节线有进展
- [ ] 伏笔记录清楚,计划回收
- [ ] 人物关系变化合理
- [ ] 整体节奏控制得当
- [ ] 无遗忘或矛盾的设定
