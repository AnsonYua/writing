# 章节创作文档模板

本模板定义了单章创作的标准格式,确保每章结构完整、质量达标。

---

## 文档命名
`chapter_XXX.md`（XXX为章节号，如：chapter_001.md、chapter_002.md）

## 文档作用

### 核心价值
- **结构标准化**:每章保持一致的质量标准
- **创作指南**:指导单章的创作流程
- **质量检查**:确保每章符合要求
- **进度追踪**:便于管理和回顾

## 章节结构建议

### 开头部分（约10-15%，300-500字）

**作用**:
- 承接上章内容
- 快速进入本章情节
- 制造悬念或冲突

**写法**:
```
✅ 好的开头:
"剑光闪过，血飞溅。一秒，生死已分。
李长青看着倒地的敌人，冷笑：'这就是赵家的顶尖高手？也不过如此。'
但他知道，真正的麻烦才刚开始。"

❌ 坏的开头:
"今天天气很好，李长青心情也不错。他决定出门散散心。
街道上人来人往，很热闹。他一边走一边想事情..."
(节奏太慢，无法吸引读者)
```

---

### 发展部分（约60-70%，1800-3500字）

**作用**:
- 推动主线剧情
- 展现角色性格
- 制造冲突和张力

**结构**:
```
1. 初始状态（300-500字）
   - 主角/世界处于什么状态
   - 引入本章的冲突或问题

2. 行动过程（1000-2000字）
   - 主角如何应对冲突
   - 遇到什么阻碍
   - 如何克服阻碍

3. 转折/高潮（500-800字）
   - 情节的转折点
   - 本章的小高潮
   - 爽点的实现
```

**注意事项**:
- 每段内容都要推动剧情
- 避免无意义的环境描写和对话
- 保持紧张感和节奏感

---

### 结尾部分（约15-20%，500-800字）

**作用**:
- 本章情节的收束
- 埋下下章的钩子
- 给读者满足感

**写法**:
```
✅ 好的结尾:
"李长青收起剑，看着远方。那里是赵家的方向。
'三个了，'他低声说，'还剩七个。'
夜风吹过，卷起地上的落叶。这一夜的杀戮，还远未结束。"

❌ 坏的结尾:
"今天就这样结束了，李长青回家睡觉了。第二天早上，他醒来..."
(平淡无奇，没有悬念)
```

**结尾钩子类型**:
1. **悬念钩**:留下未解之谜
2. **冲突钩**:新的冲突即将爆发
3. **情感钩**:角色情感发生重大变化
4. **剧情钩**:重要信息即将揭晓

## 章节类型

### 1. 剧情推进章
**特点**:推动主线剧情发展
**占比**:约70%
**重点**:
- 主线剧情占比要高
- 节奏要紧凑
- 避免支线喧宾夺主

### 2. 角色塑造章
**特点**:重点展现角色性格和成长
**占比**:约20%
**重点**:
- 通过行动展现性格，不要大量内心独白
- 角色要有成长或变化
- 也要推动主线，不能纯粹塑造

### 3. 过渡章
**特点**:连接两个重要情节
**占比**:约10%
**重点**:
- 要简洁，不要拖沓
- 必须埋下钩子吸引读者
- 可以稍微放松节奏

### 4. 高潮章
**特点**:重要情节的高潮
**占比**:不定期，但很重要
**重点**:
- 爽点要足
- 情绪要到位
- 铺垫要充分

## 常见问题

### Q1: 一章写多少字合适？
A: 建议3000-5000字。太少内容不够，太多读者疲惫。

### Q2: 一章需要包含多个情节吗？
A: 不需要。一章专注推动1-2个关键情节即可。

### Q3: 可以每章结尾都留悬念吗？
A: 不是每章都需要，但重要章节一定要有悬念钩子。

### Q4: 环境描写多少合适？
A: 不超过总字数的15%。只在必要时描写环境。

### Q5: 如何确保不水文？
A: 严格遵循"主线剧情占比≥70%"的原则，每段内容都要问自己：这对剧情有推动作用吗？

## 文件管理

### 文件命名规范
```
chapter_001.md
chapter_002.md
chapter_003.md
...
```

### 文件存储位置
```
你的小说项目/
├── novel-setting.md
├── story-outline.md
├── character-profiles.md
├── character-relationships.md
├── progress-tracking.md
├── style-guide.md
├── chapter_001.md
├── chapter_002.md
├── chapter_003.md
└── ...
```

### 章节顺序
- 按数字顺序创作（001 → 002 → 003...）
- 创作前必须阅读最近2-3章的完整内容
- 确保连贯性

---

**备注**:
- 每章创作前必须读取相关文档
- 创作后立即更新progress-tracking.md
- 发现设定变化时立即更新对应文档
- 定期（每5-10章）进行一致性检查
