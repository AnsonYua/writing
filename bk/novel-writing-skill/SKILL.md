# 小说与短剧创作 Skill

保留原有长篇网文流程，同时新增一条默认优先的“关键词直出爆款短剧”快写路径。

## 模式选择

### 模式A：长篇网文模式
在这些情况使用原有长篇流程：
- 用户明确要写长篇、连载、网文、几十章以上内容
- 用户要完整经历市场调研、风格确认、设定、大纲、角色、章节创作
- 用户要持续更新 `progress-tracking.md`

读取顺序：
- `writing-style.md`
- `plot-design.md`
- `quality-control.md`
- `consistency-check.md`
- `templates/` 下对应模板

### 模式B：短剧快写模式
在这些情况必须优先使用：
- 用户只给少量关键词，希望你直接写剧情
- 用户要“爆款短剧”“成人向短剧”“成熟向短剧”“高张力关系戏”
- 用户要分集梗概、场景拆解、开场3秒、标题钩子、前三集脚本
- 用户要“霸总、复仇、禁忌、秘密情人、逆袭、豪门、办公室”类短剧

读取顺序：
1. `references/short-drama-fast-path.md`
2. `references/tension-architect.md`
3. `references/trope-injection.md`
4. `references/title-hook-design.md`
5. `references/short-drama-script-format.md`
6. `templates/` 下短剧模板

## 短剧快写模式：核心原则

### 1. 少问，快出
- 用户只给 3 到 8 个关键词时，直接解析并补全缺失项
- 除非缺少会严重影响方向的信息，否则不要把用户卡在问答里
- 缺的信息用商业化默认值补齐，并在输出末尾简短说明假设

### 2. 默认产物不是“点子”，而是“可拍版本”
除非用户只要 brainstorm，否则首次交付默认直接输出并保存：
- `short-drama-bible.md`
- `episode-grid.md`
- `title-hook-package.md`
- `scene-001-outline.md`
- `episode-001.md`
- `episode-002.md`
- `episode-003.md`
- `short-drama-progress.md`

### 3. 成熟向写法以“张力”取胜，不靠露骨描写
- 把“成人向”理解为：禁忌感、权力差、欲望压抑、关系失控、危险亲密、道德拉扯
- 优先写眼神、距离、停顿、试探、逼近、反制、口头威胁、身份翻转
- 避免把核心爽点写成机械的露骨性描写
- 默认保持非露骨、可商业包装的成熟向表达

### 4. 包装先行
短剧不是只写剧情。必须同步设计：
- 标题
- 3到5秒停留钩子
- 每集结尾 cliffhanger
- 核心情绪曲线
- 可反复复用的关系爆点

## 关键词直出协议

当用户只给关键词时，按下列槽位理解：

| 槽位 | 说明 | 例子 |
|------|------|------|
| 题材 | 故事外壳 | 霸总 / 豪门 / 复仇 / 办公室 |
| 关系 | 核心男女关系 | 上司下属 / 前任重逢 / 假结婚 |
| 卖点 | 最强钩子 | 秘密情人 / 身份反转 / 契约关系 |
| 气质 | 情绪方向 | 高虐 / 拉扯 / 欲望感 / 打脸 |
| 结局 | 默认方向 | 追妻火葬场 / 反杀上位 / 破镜重圆 |

如果关键词不足，优先用以下商业默认值补足：
- 世界：现代都市
- 形式：24集，每集2到3分钟
- 主核：禁忌关系 + 身份反转 + 情感报复
- 节奏：开场5秒冲突，45秒升级，90秒反转，结尾强钩子
- 目标人群：爱看高张力都市情感短剧的女性受众

## 短剧快写标准流程

### 第一步：解析关键词
从用户关键词中提炼：
- 主类型
- 关系模式
- 危险点/禁忌点
- 核心爽点
- 适合的高收益套路

### 第二步：注入套路
使用 `references/trope-injection.md`：
- 选 1 个外壳套路
- 选 1 个情绪引擎套路
- 选 0 到 1 个反转套路

### 第三步：构建张力
使用 `references/tension-architect.md`：
- 先设计第一场戏的 3 分钟 beat
- 明确 hook、build-up、escalation、peak、cliffhanger

### 第四步：做短剧圣经
使用 `templates/short-drama-bible.md`：
- 书名/剧名
- 一句话卖点
- 人物关系
- 分集主线
- 视觉母题
- 短剧专用商业标签

### 第五步：做分集表
使用 `templates/episode-grid.md`：
- 每集一句话
- 每集钩子
- 每集爽点/虐点/反转点

### 第六步：做标题与开场包装
使用 `references/title-hook-design.md` 和 `templates/title-hook-package.md`

### 第七步：直接写前三集
使用 `references/short-drama-script-format.md` 和 `templates/short-drama-script.md`

## 短剧模式下的默认交付标准

### 默认规格
- 总集数：24集
- 单集时长：2到3分钟
- 单集目标：1个核心冲突 + 1个情绪升级 + 1个结尾钩子
- 前三集任务：
  - 第1集：关系撞击 + 危险吸引
  - 第2集：秘密暴露一角 + 身体/心理距离缩短
  - 第3集：第一次明确占有或反制 + 大钩子

### 每集必检
- 前 3 到 5 秒是否能停留用户
- 15 到 25 秒是否有新信息
- 45 到 90 秒是否有关系升级
- 结尾是否足以逼用户点下一集

## 写作边界与风格提醒

### 成熟向张力优先级
优先顺序：
1. 关系失衡
2. 权力拉扯
3. 欲望压抑
4. 误解与报复
5. 身份揭晓
6. 肢体逼近

### 默认避免
- 纯说明式剧情梗概
- 只有情欲、没有戏剧冲突
- 过长的背景交代
- 露骨到破坏商业包装的描写
- 所有人说话一个腔调

## 本 Skill 的真正目标

不是把故事“写完整”。

而是把故事先写成会点开、会追更、会想看下一集的样子。

用户如果只给几个词，也要优先做成能直接拍、能直接卖、能直接继续写的短剧版本。
