# 剧本自检标准

## 自检执行原则

每集剧本写完后，按以下维度逐项检查。检查结果分三类：
- **自动修复**：明显的格式错误和技术问题，无需询问用户，直接修正
- **自动优化**：质量明显不足但修法明确的问题，直接优化并告知用户
- **询问用户**：涉及创作方向、情节取舍的决策，列出来逐条询问

---

## 短剧模式自检标准

### A类：格式检查（自动修复）

| 检查项 | 标准 | 不达标处理 |
|--------|------|---------|
| 场景头格式 | `场次 X-N：内/外  地点  日/夜` | 自动补全缺失项 |
| 出场人物列表 | 每个场次都有出场人物行 | 自动添加 |
| △ 描写行数 | 每段 △ 不超过3行 | 自动拆分过长段落 |
| 台词结构 | 角色名 → 括号指示 → 台词，三行完整 | 自动补全缺失行 |
| 台词长度 | 单句不超过30字 | 自动拆分 |
| 集尾格式 | 必须有`本集完 ｜ 下集预告：`行 | 自动添加占位符，标记为待用户确认 |

### B类：节奏检查（自动优化）

| 检查项 | 标准 | 不达标处理 |
|--------|------|---------|
| 开场引爆 | 前3行内必须出现冲突或悬念 | 自动优化开场，加入冲突元素 |
| 单集字数 | 800-1200字 | 字数不足时提示用户，询问是否补充 |
| 场次数量 | 3-5个场次 | 少于3个时询问是否拆分 |
| △描写动态性 | 使用动态动词，避免静态描述 | 自动替换明显的静态描述 |
| 景别标注 | 全集至少5处景别提示 | 在关键情绪节点自动补充 |

### C类：爽点检查（询问用户）

| 检查项 | 标准 | 不达标时询问 |
|--------|------|------------|
| 爽点数量 | 每集至少2个爽点或反转 | "本集只有1个明显爽点，是否需要加强？" |
| 结尾钩子强度 | 结尾必须有强悬念 | "本集结尾钩子较弱，是否需要加强悬念？" |
| 爽点与原著匹配 | 核心爽点来自分集目录规划 | "本集爽点与目录规划有偏差，是否调整？" |

### D类：台词质量检查（自动优化+询问）

| 检查项 | 标准 | 处理方式 |
|--------|------|---------|
| 书面语台词 | 台词应口语化 | 自动优化明显的书面语 → 告知用户 |
| 废话台词 | 每句台词必须推动剧情或揭示人物 | 标记废话台词 → 询问是否删除 |
| 角色区分度 | 不同角色说话方式有差异 | 区分度不足时询问用户是否强化 |

## 动态漫模式自检标准

### A类：格式检查（自动修复）

与短剧模式相同，但字数标准调整：台词单句不超过40字。

### B类：节奏检查（自动优化）

| 检查项 | 标准 | 不达标处理 |
|--------|------|---------|
| 单集字数 | 1500-2500字 | 询问是否补充内容 |
| 场次数量 | 4-7个场次 | 少于4个时询问是否拆分 |
| 开场引爆 | 前5行内出现冲突或悬念（略慢于短剧） | 自动优化 |
| 情感描写 | 允许适当的内心描写（旁白形式） | 检查旁白是否过多（不超过全集20%） |

### C类：爽点检查（询问用户）

| 检查项 | 标准 | 不达标时询问 |
|--------|------|------------|
| 爽点数量 | 每集至少1个强爽点（允许更多铺垫） | "本集情绪峰值不明显，是否需要加强？" |
| 对白节奏 | 对话来回不超过6轮不推进剧情 | "本处对话较拖，是否压缩？" |

## 自检报告输出格式

自检完成后，向用户展示：

```
📋 第 N 集自检报告

【已自动修复】（X项）
✅ 第2场台词过长，已拆分为2句
✅ 第3场缺少出场人物，已补充
✅ 第1场 △ 描写超过3行，已精简

【已自动优化】（X项）
🔧 开场节奏稍慢，已在第一行加入冲突元素
🔧 第4场有2句书面语台词，已调整为口语化

【需要您确认】（X项）
❓ 本集爽点数量为1个，建议至少2个。是否在第3场加强反派被打脸的力度？[是/否]
❓ 本集结尾钩子：「他看向窗外，沉默不语。」力度偏弱。是否改为更强的悬念？[是/否/我来写]
❓ 第2场台词「吾等在此恭候多时」语感偏古代，本剧设定为现代。是否修改？[是/否]

请逐条回复，或输入「全部保持」跳过所有询问。
```

## 跨集一致性专项检查（/一致性）

当用户执行 `/一致性` 指令时，或完成每5集时自动触发一次。

### 检查维度

**角色一致性**
- 角色名称和称呼是否前后统一（对照角色档案.md）
- 角色能力/身份是否出现前后矛盾
- 角色关系变化是否有合理过渡

**道具/信息一致性**
- 重要道具是否在集与集之间前后呼应
- 关键信息（时间线、地点）是否连贯
- 已揭露的秘密是否在后续有所体现

**伏笔追踪**
- 已埋设的伏笔是否按计划回收
- 是否有遗漏未处理的伏笔
- 伏笔回收时机是否合理

**输出格式**

```
🔍 一致性检查报告（第1-N集）

【无问题项目】
✅ 角色称呼：全部一致
✅ 时间线：逻辑通顺

【发现问题】
⚠️ 第3集主角称呼反派为「王总」，第5集改称「老王」，建议统一
⚠️ 第2集埋设的伏笔（主角保险箱里的文件）至第X集尚未回收

是否需要逐一修正？[是 / 跳过]
```
