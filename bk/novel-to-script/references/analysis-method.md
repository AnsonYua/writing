# 小说分析方法论

## 分析目标

将小说原文转化为改编蓝图：找出哪些内容必须保留（核心爽点、关键转折），哪些可以压缩（铺垫、世界观说明、日常过渡），哪些需要强化（冲突、对抗、情绪爆发点）。

---

## 一、题材识别与风格定位

拿到原文后，首先判断：

| 题材类型 | 识别关键词 | 改编重点 |
|---------|---------|---------|
| 都市爽文（霸总/战神/重生） | 总裁、公司、逆袭、归来 | 身份碾压、打脸、财力展示 |
| 玄幻修仙（系统/升级/废柴） | 修炼、境界、丹药、系统 | 实力碾压、突破升级、战斗场面 |
| 古代言情/权谋 | 皇宫、朝堂、娘娘、嫡庶 | 权谋反转、情感虐恋、身份揭露 |
| 现代言情 | 暗恋、误会、前任、婚姻 | 情感张力、误会解除、情绪爆发 |
| 悬疑推理 | 案件、凶手、证据、真相 | 线索铺设、真相揭露、反转震撼 |
| 重生穿越 | 重来一次、前世、穿越 | 先知优势、改变命运、复仇爽感 |

---

## 二、爽点提炼方法

### 爽点类型识别

**身份碾压型**（最常见，最容易改编）
- 特征：主角隐藏真实身份/实力，在被轻视后揭露
- 识别词：\"你知道他是谁\"\"这是我的人\"\"全场震惊\"
- 改编策略：强化揭露瞬间，增加围观群众的震惊反应

**打脸复仇型**（爽感最直接）
- 特征：对方刚嘲笑/欺负主角，立刻被现实打脸
- 识别词：\"话音刚落\"\"没想到\"\"当场傻眼\"
- 改编策略：让反派先嚣张够，再让打脸来得更猛

**实力碾压型**（视觉冲击强）
- 特征：战斗、比拼、竞争中主角以碾压姿态获胜
- 识别词：\"一招\"\"秒杀\"\"毫无还手之力\"
- 改编策略：加入景别设计，强调动作画面感

**情感爆发型**（女频必备）
- 特征：压抑已久的情感在关键时刻爆发
- 识别词：\"终于\"\"再也忍不住\"\"你知道吗\"
- 改编策略：用台词和特写镜头放大情绪峰值

**逆袭翻盘型**（男频核心）
- 特征：从最低谷一步步反击，每次升级都是爽点
- 识别词：\"从今天起\"\"我回来了\"\"这一切都将改变\"
- 改编策略：每集安排一个小逆袭，积累到大逆袭

### 爽点强度评级

| ⭐ | 说明 |
|----|------|
| ⭐⭐⭐⭐⭐ | 全书高潮，必须保留并强化，建议单集重点呈现 |
| ⭐⭐⭐⭐ | 重要爽点，需保留，可适当强化 |
| ⭐⭐⭐ | 一般爽点，根据集数决定是否保留 |
| ⭐⭐ | 铺垫性内容，可压缩为旁白或闪回 |
| ⭐ | 过渡性内容，可删除 |

## 三、转折点分析

转折点是情节方向发生重大变化的时刻，必须全部保留。

### 识别方法

- **人物关系转折**：敌→友、友→敌、爱→恨
- **信息揭露转折**：隐藏的真相被揭开（身世、阴谋、背叛）
- **力量对比转折**：弱→强、劣势→优势
- **情感转折**：冷漠→心动、误解→理解

### 转折点处理原则

- 每个重大转折最好单独成集（或作为一集的核心收尾）
- 转折前要有足够铺垫（但改编时可以压缩铺垫，保留转折）
- 转折后要有即时反应（角色情绪、局势变化）

---

## 四、可删减内容判断

以下内容在改编时可以大幅压缩或删除：

| 内容类型 | 处理方式 |
|---------|---------|
| 世界观详细说明 | 改为角色台词一句带过，或字幕说明 |
| 修炼/升级的具体过程 | 只保留结果（\"三天后，他突破了\"） |
| 不推进主线的日常场景 | 直接删除 |
| 超过2章的铺垫段落 | 压缩为1个场景或旁白 |
| 次要人物的大量戏份 | 合并角色，或只保留关键交互 |
| 重复的情感表达 | 只保留最高峰的一次 |

---

## 五、角色合并与简化原则

原著中的次要角色在改编时可以合并：

- 多个反派→合并为1-2个核心反派
- 多个支持者→合并为1-2个核心帮手
- 工具性角色→只在需要时出现，不单独发展

**合并原则**：功能相似的角色合并，确保每个出场角色都有推动剧情的作用。

---

## 六、集数估算逻辑

### 基础估算

短剧模式（每集2-3分钟，约800-1200字剧本）：
- 密集冲突章节：每章 → 0.5-1集
- 普通章节：每2-3章 → 1集
- 铺垫章节：每4-5章 → 1集

动态漫模式（每集3-5分钟，约1200-2000字剧本）：
- 整体比短剧多20-30%的集数

### 特殊调整

- 高密度爽点段落：可适当拉慢，每个爽点单独处理
- 低密度铺垫段落：大幅压缩，多章合并为1集
- 高潮章节：建议拆分成2-3集，充分展现
- 结局章节：最后2-3集专门处理大结局

### 给用户的建议格式

```
根据原文分析，推荐以下方案：

📺 短剧标准版：约 X 集（每集2-3分钟）
   适合：快节奏，高爽感，适合短视频平台
   
📖 动态漫版：约 X 集（每集3-5分钟）
   适合：有更多对话和情感描写空间

🎯 精华版：约 X 集（重点提炼）
   适合：只保留核心爽点，极致压缩

请选择方案，或告诉我你的目标集数（我会按此调整压缩比例）：
```

## 七、角色档案提炼方法

分析阶段同步提炼角色档案，保存为 `角色档案.md`，供写集时引用，防止跨集人物设定不一致。

### 主要角色提炼标准

提炼原则：只记录对改编有直接影响的信息，不做小说式的详细描写。

**必须记录的信息：**
- 姓名（含外号、别称）
- 核心身份（隐藏身份也要记录）
- 与主角的关系
- 性格关键词（2-3个）
- 说话风格特点（台词写作参考）
- 称呼规范（自称、他称、特殊场景称呼）

**角色分级：**
- **核心角色**（主角、主要反派、主要配角）：完整档案
- **次要角色**（功能性角色）：姓名+关系+功能一句话说明
- **可合并角色**：注明"建议与XX合并"

### 称呼规范表（防止跨集不一致的关键）

```markdown
## 称呼规范表

| 角色 | 自称 | 主角对其称呼 | 其他角色对其称呼 | 特殊场景 |
|------|------|------------|--------------|---------|
| 陈锋（主角） | 我 | - | 陈总（揭露后）、废物（揭露前） | 下属称：龙主 |
| 林母 | 我、本人 | 岳母、阿姨 | 林太太 | 正式场合：林夫人 |
```

## 八、原著章节追踪表建立方法

章节追踪表是防止转换遗漏或重复处理原著章节的核心工具。

### 建立步骤

**第一步**：将原著按内容密度分段（不是按章节号机械切分）

分段原则：
- 找出情节自然断点（一个冲突结束、场景切换）
- 根据爽点分布调整（高密度爽点段落单独成组）
- 参考集数规划，让每组大致对应1集内容

**第二步**：为每组撰写一句话内容摘要

摘要要点：
- 包含本段最重要的爽点或事件
- 包含关键出场人物
- 标注是否含有高强度爽点（加⭐标记）

**第三步**：初始状态全部标记为 ⬜待转，随写集进度更新

### 追踪表格式

```markdown
## 原著章节追踪表

| 组号 | 章节范围 | 字数 | 主要内容摘要 | 核心爽点 | 对应集数 | 状态 |
|------|---------|------|------------|---------|---------|------|
| 1 | 第1-3章 | 约8000字 | 主角被羞辱，获得系统，初露实力 | 系统激活⭐⭐⭐ | 第1-2集 | ⬜待转 |
| 2 | 第4-5章 | 约5000字 | 公司第一次对抗，财力初展示 | 打脸场景⭐⭐⭐⭐ | 第3集 | ⬜待转 |
```

### 状态标记说明

| 状态 | 含义 |
|------|------|
| ⬜待转 | 尚未处理 |
| 🔄转换中 | 当前正在写对应集数 |
| ✅已转 | 已完成转换 |
| ⏭️已跳过 | 内容已合并入相邻集或删除 |
