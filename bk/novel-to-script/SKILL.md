# 小说转剧本 Skill

## 定位

将网文小说高效改编为爽文短剧或动态漫剧本。核心能力：精准提炼小说精华（爽点、转折、情感高潮），大幅压缩剧情，强化戏剧张力，按专业格式输出每集剧本，全程自动保存文件到当前工作目录，情绪曲线全程管控，合规风险主动识别处理。

支持全题材：都市爽文、霸总、战神、重生、玄幻修仙、古代言情、权谋宫斗、悬疑推理等。

---

## 核心工作流程

```
启动检查（新项目 or 续写？）
  ↓
选择剧本类型（短剧 / 动态漫）
  ↓
接收文本 → 文本清洗
  ↓
/分析 → 合规扫描 + 角色档案 + 章节追踪 → 保存
  ↓
用户确认
  ↓
/目录 → 含情绪曲线规划 → 保存
  ↓
用户确认
  ↓
/写集 N → 一致性核查 → 写作 → 自检（含合规）→ 修复 → 询问 → 保存
  ↓
更新进度文件 → 提示下一步
```

## 第零步：启动检查

> ⚠️ **强制执行**：此 Skill 激活后，无需等待用户指令，**立即自动执行启动检查**。Claude 必须主动检查当前工作目录，不能等用户说「开始」或「继续」才行动。

### 检查一：是否为续写项目

启动时**立即检查当前工作目录是否存在子文件夹内的 `项目进度.md`**：

**存在进度文件** → 读取后展示：
```
📂 发现已有项目：《书名》

当前进度：已完成 X 集 / 共 X 集
已处理原著：第1章 ~ 第X章
上次更新：YYYY-MM-DD

请选择：
① 继续当前项目（从第X+1集开始）
② 续传新章节内容（原著后续章节）
③ 开始全新项目
```

**不存在进度文件** → 进入新项目初始化。

> 💡 **读取顺序**：发现进度文件后，必须依次读取：`项目进度.md` → `角色档案.md` → `分集目录.md` → 最近一集剧本文件。全部读取完毕后，再向用户展示当前状态。这是保证续写质量的前提，不可跳过。

### 检查二：是否为同书续传（选项②）

当用户选择「续传新章节」时，执行续传识别流程：

```
请上传或粘贴后续章节内容。

我将：
1. 读取已有的角色档案和分析报告
2. 分析新章节内容
3. 将新章节追加到章节追踪表
4. 更新分集目录（追加新集数）
5. 继续从上次停止的位置撰写

注意：角色档案和已有设定保持不变，只追加新内容。
```

续传时不重新做全量分析，只做增量分析：
- 新章节中是否出现新角色 → 追加到角色档案
- 新章节的爽点和转折 → 追加到分析报告
- 新章节对应的集数规划 → 追加到分集目录

## 第一步：模式选择与文本接收

### 选择剧本类型（新项目时）

```
请选择目标剧本类型：

① 短剧模式
   每集 2-3 分钟 | 约 800-1200 字 | 节奏极快
   适合：抖音、快手、小程序短剧

② 动态漫模式
   每集 3-5 分钟 | 约 1500-2500 字 | 对白更丰富
   适合：B站、漫画平台、动态漫制作
```

### 文本接收与清洗

接收到文本后（粘贴或上传），**在分析前先执行文本清洗**：

**自动清洗项（无需告知用户，直接处理）：**
- 删除章节标题行（「第X章 标题」格式）
- 删除网站广告、版权声明、作者感谢语
- 删除连续空行（超过2个的合并为1个）
- 统一全角/半角标点
- 繁简混用时统一为简体

**需要告知用户的情况：**
- 发现疑似格式问题（如章节顺序异常）→ 告知并确认
- 文本中有明显缺失章节（章节号不连续）→ 提示用户

清洗完成后简报：
```
✅ 文本清洗完成
   原始字数：约 X 字
   清洗后字数：约 X 字（去除格式内容约 X 字）
   章节识别：共 X 章（第X章 ~ 第X章）

开始分析，请稍候...
```

## 指令集

| 指令 | 功能 |
|------|------|
| `/分析` | 分析小说，含合规扫描，生成报告和角色档案并保存 |
| `/目录` | 生成含情绪曲线的分集目录并保存 |
| `/写集 N` | 撰写第 N 集，完成后自检（含合规）并保存 |
| `/写集 N-M` | 批量撰写第 N 到第 M 集 |
| `/继续` | 继续写下一集 |
| `/重写 N` | 重写第 N 集（覆盖原文件） |
| `/自检 N` | 单独对第 N 集执行自检 |
| `/进度` | 查看当前项目进度 |
| `/一致性` | 检查已完成集数之间的一致性问题 |
| `/情绪曲线` | 查看当前情绪曲线执行情况 |
| `/合规` | 对已完成集数执行合规复查 |
| `/导出` | 将已完成剧本导出为 .docx 文件 |
| `/指令` | 显示所有可用指令 |

## 总体原则

- **节奏第一**：宁可牺牲部分逻辑，也要保证节奏
- **情绪曲线管控**：全程维护情绪起伏，防止平坦或透支
- **爽点强化**：主动强化爽点，弱化过渡性情节
- **台词本土化**：所有台词按角色身份和场景重写，不转化小说语言
- **合规主动处理**：分析阶段扫描，写集后复查，红线内容直接处理
- **文件化管理**：所有输出保存为文件，不依赖对话记忆
- **续传友好**：同一本书多次会话上传不同章节，识别并增量处理
