Fax API Vendor Check: Fax.Plus vs eFax

Prepared for vendor sourcing. Date: 21 Apr 2026.

Requirement

The vendor must provide an API to send a fax to another party. After submission, we must be able to check whether the send job succeeded or failed, meaning whether the fax was delivered to the recipient fax endpoint.

This does not prove that a human opened or read the fax. It only confirms technical fax transmission to the receiving fax system.

Summary

Vendor Can send fax by API? Can check delivery success/failure? Finding
Fax.Plus Confirmed Confirmed Public API docs show the send endpoint, fax record endpoint, JSON status field, status values, and webhook payload.
eFax Corporate / Enterprise Confirmed Not fully confirmed from public docs Public docs confirm fax submission, sent-fax list, metadata retrieval, and outbound webhooks. However, public docs do not show the exact JSON response for GET /faxes/{fax_id}/metadata, so the exact field/value proving delivery is not confirmed.

Fax.Plus Finding

1. Sending a fax

Fax.Plus confirms this endpoint:

POST /accounts/{user_id}/outbox

The send API can return destination-to-fax-ID mapping when return_ids=true.

{
  "ids": {
    "+1234567890": "1a2b3c4d5e6f7890",
    "+1345678912": "78901a2b3c4d5e6f"
  }
}

2. Checking delivery result

Fax.Plus confirms this endpoint:

GET /accounts/{user_id}/faxes/{fax_id}

The public docs show a fax record response with a status field.

{
  "id": "5e7de3ad54cfd54eb568cc76",
  "direction": "outgoing",
  "from": "+12076001783",
  "to": "+1234567890",
  "pages": 2,
  "status": "success",
  "description": "OK",
  "start_time": "2020-03-27 11:29:21",
  "last_update": "2020-03-27 11:29:49"
}

Fax.Plus also publicly documents possible status values. success means the fax was successfully sent. Failure values include failed, failed_user_busy, failed_no_answer, failed_unallocated_number, and several fax transport errors such as failed_fs_31 and failed_fs_35.

3. Webhook option

Fax.Plus confirms a fax_sent webhook. The public webhook example includes status: "success".

{
  "hook": {
    "event": "fax_sent",
    "event_time": "2024-09-09 10:44:39"
  },
  "data": {
    "id": "66df09cf1d6d0b295c363956",
    "pages": 1,
    "from": "+1 334-304-xxxx",
    "to": "+1 334-304-xxxx",
    "status": "success"
  }
}

Conclusion for Fax.Plus: Fax.Plus satisfies the requirement. The send API and delivery status check are both publicly documented.

eFax Corporate / Enterprise Finding

1. Sending a fax

eFax public documentation confirms this endpoint:

POST /faxes

This supports the requirement to submit an outbound fax by API.

2. Checking sent fax and metadata

eFax public documentation confirms these endpoints:

GET /faxes/sent
GET /faxes/{fax_id}/metadata

The main eFax API page says the Fax Services API supports fax transmissions, downloading metadata, webhooks for inbound and outbound fax notifications, and retrieving lists of sent and received faxes.

3. Delivery success/failure detail

The eFax Developer Product Overview says that after completion, eFax posts back fax results including transmission identifier, fax number, completion date/time, fax status, recipient CSID, duration, number of pages, and number of retries. It also says transaction status can be queried at any time.

Important: I did not find a public eFax page showing the exact JSON response for GET /faxes/{fax_id}/metadata. Therefore, I cannot confirm the exact JSON field name, exact status values, or the exact value that means delivered.

Conclusion for eFax: eFax likely supports the required workflow, but from public documentation alone, delivery-status JSON is not fully confirmed. Before selecting eFax, ask eFax to provide the official response schema and sample JSON for GET /faxes/{fax_id}/metadata, including one successful delivery and one failed delivery example.

Questions to Send to eFax

To close the gap, send these questions to eFax:

  1. Please provide the official JSON response schema for GET /faxes/{fax_id}/metadata.
  2. Which response field confirms final delivery result?
  3. What exact value means the fax was delivered to the recipient fax endpoint?
  4. What are all possible status values?
  5. Please provide one sample JSON for successful delivery and one sample JSON for failed delivery.
  6. Please provide the outbound webhook/postback sample payload for fax completion.

Recommendation

Fax.Plus is the safer choice based on public technical evidence because it publishes the send endpoint, status endpoint, status field, status values, and webhook payload.

eFax should not be rejected yet, but its public documentation is not enough to confirm the exact delivery-status JSON. Treat eFax as conditionally acceptable until their API team provides the missing schema and sample responses.
