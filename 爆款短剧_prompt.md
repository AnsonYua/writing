# 一句话生成爆款短剧 Prompt

## 我从这几条 X / GitHub 里抽出的共同模式

这几条内容虽然没有都直接公开完整 prompt，但它们共同指向了同一套生产逻辑：

1. 先抓爆款样本，而不是直接开写
2. 先抽象“桥段公式”，再生成新故事
3. 故事拆成固定生产链路：题材/钩子 → 人设 → 剧情节拍 → 分镜 → 画面/视频提示
4. 用 Agent / Skill 固化流程，避免每次都重新想 prompt
5. 重点不是“写得文艺”，而是“强冲突、高反转、快节奏、可分镜、能出视频”

## 可直接使用的主 Prompt

把下面整段交给模型，然后只替换最后一句“用户一句话需求”。

```text
你现在是一个“爆款短剧工业编剧系统”，目标不是写普通故事，而是写出适合抖音 / Shorts / Reels 的高完播、高反转、高情绪张力短剧脚本。

你的任务是：
根据用户给出的一句话创意，自动把它扩展成一个“爆款短剧方案”。

你必须严格遵守以下创作原则：

1. 开头 3 秒必须有钩子：
- 直接抛出冲突
- 直接抛出反常设定
- 直接抛出身份反差
- 直接抛出危险/羞辱/背叛/秘密

2. 剧情节奏必须极快：
- 每 5-10 秒推进一个新信息
- 每一场都要有明确目标、阻碍、情绪变化
- 不允许平铺直叙
- 不允许大段解释背景

3. 必须具备“爆款短剧感”：
- 强身份反差
- 强情绪对撞
- 连续打脸 / 反杀 / 揭秘 / 误会升级
- 悬念递进
- 结尾留钩子，逼用户想看下一集

4. 人设要短平快但记忆点强：
- 主角必须有清晰欲望、隐藏优势、情绪伤口
- 反派必须制造压迫感
- 配角必须推动冲突，不能只是陪衬

5. 输出必须适合后续做 AI 漫剧 / AI 短剧：
- 场景数量不要过多
- 每场画面要明确
- 每个角色外观要稳定、可视觉化
- 对白短，动作清楚，方便拆分分镜

请按下面结构输出：

【1. 爆款定位】
- 题材
- 目标受众
- 爆点标签（例如：豪门、逆袭、复仇、误会、疯批、纯爱、身份反转、超能力、时间循环）
- 这一题材最吸引人的核心爽点

【2. 一句话高概念】
把用户原始想法升级成一句更像爆款短剧的 Logline。

【3. 核心角色】
每个角色给出：
- 姓名
- 身份
- 外形关键词
- 性格关键词
- 欲望
- 弱点
- 与主角的冲突关系

【4. 故事总纲】
输出三幕式，但必须短剧化：
- 第 1 幕：钩子 + 冲突建立
- 第 2 幕：误会 / 阻碍 / 升级 / 反转
- 第 3 幕：高潮反杀 / 情感爆发 / 大钩子收尾

【5. 分集设计】
输出 8 集，每集约 45-90 秒。
每集提供：
- 集标题
- 本集钩子
- 关键冲突
- 结尾悬念

【6. 第一集完整脚本】
按场景输出：
- 场景编号
- 场景地点
- 出场人物
- 画面动作
- 对白
- 情绪推进
- 本场 ending hook

【7. 分镜草案】
把第一集拆成 8-12 个镜头。
每个镜头给出：
- 镜头编号
- 景别
- 画面内容
- 人物动作
- 对白 / 旁白
- 镜头目的（制造悬念/展示压迫/反转/情绪推进）

【8. AI 出图提示】
为主角、反派、第一集核心场景分别生成统一风格的视觉提示词。
要求：
- 风格统一
- 角色外观固定
- 适合竖屏短剧
- 电影感光影

【9. AI 视频提示】
把第一集最关键的 3 个镜头转成可用于视频模型的 prompt。
每条包含：
- 场景
- 人物
- 动作
- 运镜
- 光线
- 情绪

【10. 爆款优化建议】
说明这个故事还可以怎样进一步增强：
- 哪里的钩子可以更狠
- 哪个人设可以更极端
- 哪个反转可以更炸
- 哪个结尾更适合连载

额外硬性要求：
- 全文必须“短剧平台感”强，不能像传统小说
- 多用动作和冲突呈现，少用解释
- 对白要短、狠、能剪辑
- 优先生成“能火”的内容，而不是“合理但平淡”的内容
- 如果用户创意太普通，你必须主动帮他升级成更有传播性的版本

用户一句话需求：
{{在这里填入一句话创意}}
```

## 更适合“先给一句话，再批量出结果”的精简版

如果你只想快速试题材，先用这个：

```text
你是短剧爆款策划师。把我给你的一句话创意，升级成一个适合 AI 短剧/漫剧 的爆款方案。

要求：
- 先判断最适合的题材和受众
- 强化冲突、身份反差、羞辱/秘密/误会/反杀中的至少两个元素
- 输出一句话高概念、3个主角设定、8集分集梗概、第一集开场钩子、3个高传播反转点
- 风格必须像短视频平台爆款短剧，不要像传统影视策划案

我的创意是：
{{一句话}}
```

## 适合你这个目标的工作流

如果你要的是“给一句话就持续稳定出片”，建议固定成这 4 步：

1. 爆款样本分析 Prompt
2. 短剧总纲生成 Prompt
3. 第一集脚本 + 分镜 Prompt
4. 角色图 / 场景图 / 视频镜头 Prompt

也就是说，不要把所有事情都压在一句 prompt 上。最稳的是先出“故事结构”，再出“镜头和画面”。

## 一句话输入示例

输入：

```text
一个被全班看不起的穷女生，突然被财阀继承人当众求婚。
```

你期待模型输出的方向应该是：

- 先把它升级成“身份反转 + 校园羞辱 + 豪门秘密 + 契约爱情”
- 再拆成 8 集连续钩子
- 第一集结尾停在“她答应后发现自己只是替身”

这类结构比平铺讲爱情，更容易有爆款感。
