# Short Drama Scene Pack Review

Review an existing scene pack like a strict upstream production editor.

This skill is for review, not regeneration. Keep the storyline locked.

## Hard Boundary

Do not change the storyline.

That means:

- do not add new plot events
- do not move reveals earlier
- do not change who knows what
- do not merge scenes if the merge would alter suspense timing
- do not invent new props, rooms, or confrontations

You may:

- flag duplicated scene function
- flag weak scene boundaries
- flag abstract emotional shorthand
- flag non-shootable visible action
- suggest safer scene compression or clarification

## Main Goal

Judge whether the scene pack is ready for storyboard and image stages.

It should:

- split scenes at motivated dramatic boundaries
- keep one dominant function per scene
- describe visible action in shootable terms
- protect continuity into later episodes
- avoid repeating the same proof, fear, or pressure scene twice

## Core Questions

For each scene, ask:

1. What is this scene's one dominant dramatic job
2. Does the next scene do a genuinely new job, or just repeat it
3. Is the scene boundary motivated by space, pressure, prop control, or tactical change
4. Is `visible action` actually playable on screen
5. Has abstract emotional shorthand been translated into visible body, prop, and space behavior
6. Is the proof object named correctly and naturally
7. Does this scene introduce a new room that downstream generation will need a matching room anchor for
8. Does the end hook actually hand off into the next scene
9. If this scene were merged with the next one, would we lose anything real
10. If this scene were removed, would the story still read the same

## Red Flags

Mark as problems when the scene pack:

- splits a new scene only because the proof gets deeper
- repeats the same dramatic function across adjacent scenes
- uses visible action like `她卡住了`, `她不敢相信`, `氣氛變緊`, `他表面正常` without showing body or prop behavior
- writes emotional conclusion instead of stageable action
- uses technical proof wording that sounds like camera geometry, not first-shock recognition
- creates a scene whose main job already finished in the previous scene
- keeps a weak bridge scene that only delays the stronger next turn
- makes character presence feel illogical from one scene to the next
- moves into a clearly new room, but gives no downstream warning that the generation stack may need a new matching room anchor

## Abstract Emotion Translation Standard

Whenever a scene uses emotional shorthand, ask whether it can be rewritten into drawable, actable behavior.

Weak:

- `她整個人先卡住`
- `曉晴不敢相信`
- `李美珍很害怕`

Stronger:

- `她手裡的充電線停在插口前，肩膀僵住，眼睛釘在螢幕上`
- `她盯住母親，像等對方立刻處理，卻只等到一句更低的聲音`
- `李美珍先看門，再把手機按黑，握住手機的手指節慢慢發白`

If the reviewer can understand the feeling but still cannot imagine the body and prop state, mark it weak.

## Review Method

Follow this order:

1. Identify the locked storyline
2. Identify each scene's dominant job
3. Check whether adjacent scenes duplicate function
4. Check whether scene boundaries are motivated
5. Check whether visible action is truly shootable
6. Check whether emotional shorthand has been translated into visible staging
7. Check whether any new room should trigger a downstream room-anchor warning
8. Check proof wording
9. Check continuity handoff
10. Produce findings

## Output Format

Use this structure:

### Review Summary

- material reviewed
- overall scene efficiency: strong / medium / weak
- overall shootability: strong / medium / weak
- biggest risk before storyboard

### Scene-by-Scene Review

For each scene, include:

- scene id or label
- dominant function
- verdict: strong / medium / weak
- boundary logic: pass / warning / fail
- visible action quality: pass / warning / fail
- duplication risk
- continuity risk
- issue
- safe fix

### Top Problems

List the highest-priority repeated issues.

Use seriousness labels:

- serious
- medium
- light

### Safe Upgrade Notes

Give 3-5 safe upgrades that do not change storyline.

Prefer upgrades like:

- merge duplicated scenes
- rewrite visible action into body or prop behavior
- make hook image more concrete
- make proof wording more human-recognition-first
- warn early when a new room will need a real matching anchor instead of relying on scene-number naming

## Tone

Be practical, strict, and upstream-production minded.

Do not rewrite the plot.
Do not praise weak scene design just to be polite.
